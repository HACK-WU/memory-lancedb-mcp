## 1. 设计原则

1. **快速失败**：输入校验失败立即报错退出，不静默降级
2. **自动恢复**：运行时数据损坏尽量自动修复，无需人工介入
3. **最小暴露**：自动恢复的错误不暴露给 AI Agent，仅记录日志
4. **幂等安全**：重复操作不产生副作用（如重复向量化、重复导入）

## 2. 通用参数校验

| 场景 | 行为 | 暴露 |
|------|------|------|
| `--scope` 未指定 | 报错退出："必须通过 --scope 指定项目 scope" | 是 |
| `--scope` 含非法字符 | 报错退出："scope 仅允许字母、数字、连字符、下划线" | 是 |
| `--scope` 含路径遍历 `../` | 报错退出："scope 不允许包含路径遍历字符" | 是 |

## 3. Group 树索引相关

| 场景 | 行为 | 暴露 |
|------|------|------|
| Group 路径不存在 | 返回空 `hot_relations` + 空 `word_cloud`；不崩溃 | 是 |
| group-index.json 不存在 | 自动从 `_template/` 复制初始化 | 否 |
| group-index.json 损坏/格式错误 | 报错退出 + 损坏文件路径；建议"从备份恢复或重新初始化" | 是 |
| 新 scope 首次使用 | 自动从 `_template/` 复制初始化文件；自动创建目录 | 否 |
| manage-index 创建已存在的 Group | 报错退出："Group 已存在" | 是 |
| manage-index 删除含子节点的 Group | 报错退出："请先删除子节点" | 是 |

## 4. Relations 缓存相关

| 场景 | 行为 | 暴露 |
|------|------|------|
| relations-cache.json 损坏 | 降级：跳过缓存层，所有查询直接走检索路径；下次写入覆盖损坏文件 | 否 |
| Relation ID 在本地 KB 中不存在 | 返回 null + 提示"请在记忆系统中通过关键词检索" | 是 |
| 淘汰 Relation 时无 keywords 可提取 | 丢弃该 Relation，不生成关键词 | 否 |
| sync-relation 传入空 module-info | 报错拒绝写入："模块信息不能为空" | 是 |
| sync-relation keywords 中有不存在于原文的词 | 移除该词，放入 `invalid_keywords` 返回，仍继续写入 | 是（警告） |
| sync-relation keywords 为空数组 | 写入 relation 但 keywords 为空，发出警告 | 是（警告） |
| sync-relation 批量模式中某条 group 不存在 | 跳过该条，记录到 `failed`，不中断其余条目 | 是 |
| 冷热分区数量超限 | 将最低评分数据转移到次级分区，无次级分区则删除 | 否 |

## 5. 本地 KB 相关

| 场景 | 行为 | 暴露 |
|------|------|------|
| 本地 KB JSON 损坏/格式错误 | 返回错误信息 + 损坏文件路径；建议"从记忆系统同步恢复" | 是 |
| 本地 KB + 记忆系统均未命中 | AI 暂停并提示用户知识缺失，请求提供线索 | 是 |
| 记忆系统检索结果与需求不匹配 | 视同未命中，AI 暂停并提示用户补充知识 | 是 |

## 6. 评分机制相关

| 场景 | 行为 | 暴露 |
|------|------|------|
| useCount 溢出 | 限制最大值 10（`maxUseCount`），超过不再增加 | 否 |
| lastUsedTime 为未来时间戳 | 视为当前时间处理，记录警告 | 否 |
| 评分数据损坏 | 重置 useCount=0、lastUsedTime=null，记录警告 | 是（警告） |
| partition_config 配置错误 | 使用默认配置，记录警告 | 是（警告） |
| 分区转移失败 | 记录错误，不影响正常使用 | 是（错误） |
| 新兴热门席位不足 | 按最后使用时间排序，优先保留最新的 | 否 |

## 7. 外部知识库导入相关

| 场景 | 行为 | 暴露 |
|------|------|------|
| import-kb --source 目录不存在 | 报错退出："源目录路径不存在" | 是 |
| import-kb --root-name 与已有根节点重名 | 报错退出："根节点已存在，请使用不同名称" | 是 |
| import-kb 映射文件中引用的文件不存在 | 跳过该条，记入 `errors`，继续处理其余 | 是 |
| import-kb .md 文件内容为空 | 跳过该条，记入 `errors` | 是 |
| import-kb 重复导入同一目录 | 幂等：已存在的 Relation 覆盖更新，不产生重复 | 否 |
| import-kb 超大文件（>10MB） | 直接跳过，不导入，记录警告 | 是（警告） |
| import-kb 特殊字符处理 | 直接删除特殊字符，记录警告 | 是（警告） |
| import-kb 并发导入冲突 | 使用拆分合并机制：每个 Agent 单独创建索引，完成后合并 | 否 |
| import-kb 未提供 --scan-index | 正常导入，关键词为空，不影响本地 KB 功能 | 否（警告） |

## 8. 预扫描相关

| 场景 | 行为 | 暴露 |
|------|------|------|
| scan-kb --source 目录不存在 | 报错退出："源目录路径不存在" | 是 |
| scan-kb .md 文件内容为空 | 跳过该文件，记入 `warnings` | 是（警告） |
| scan-kb AI 摘要生成失败 | 跳过该文件，记入 `errors`，继续处理其余 | 是 |
| scan-kb vectorize 时 scan-index.json 不存在 | 报错退出："请先执行 scan 子命令" | 是 |
| scan-kb vectorize 单条向量化失败 | 记入 `errors`，继续处理其余，该条 `vectorized` 保持 `false` | 是 |
| scan-kb vectorize 时记忆系统不可用 | 报错退出，不修改 scan-index.json 状态 | 是 |
| 摘要格式错误（无路径标记） | 跳过该条，记入 `errors` | 是 |
| 路径标记格式错误 | 跳过该条，记入 `errors` | 是 |
| 路径对应的文件不存在 | 跳过该条，记入 `errors` | 是 |

## 9. WAL 写入相关

| 场景 | 行为 | 暴露 |
|------|------|------|
| 临时文件写入失败 | 报错退出，原文件不变，提示"写入失败，请检查磁盘空间和权限" | 是 |
| rename 失败 | 报错退出，原文件不变，临时文件残留 | 是 |
| 临时文件残留 | 下次启动时自动清理 `*.tmp` 文件 | 否 |
| 并发写入同一文件 | 应用层保证串行（脚本同步执行）；不同 scope 天然隔离 | 否 |

## 10. 记忆系统交互相关

| 场景 | 行为 | 暴露 |
|------|------|------|
| 记忆系统不可用（MCP 超时/错误） | 返回"记忆系统不可用，请稍后重试"；不写入损坏数据 | 是 |
| memory_store 成功但 scan-index.json 更新失败 | 下次 vectorize 时该条目仍为 `false`，会重复调用 memory_store（幂等覆盖） | 否 |
| Agent 发现记忆不存在 | 自动触发同步机制，从记忆系统或本地 KB 同步数据 | 否 |

## 13. 增量扫描相关

| 场景 | 行为 | 暴露 |
|------|------|------|
| git 命令执行失败 | 退化为全量扫描，输出 warning | 是（警告） |
| lastScannedCommit 不存在（如 rebase 后） | 退化为全量扫描，输出 warning | 是（警告） |
| git diff 输出解析异常 | 退化为全量扫描，记录错误日志 | 是（警告） |
| 增量扫描中 D 类文件已不存在但 scan-index 有记录 | 正常执行 memory_forget + 清理 scan-index 条目 | 否 |
| 增量扫描中 M 类文件读取失败 | 跳过该文件，记入 `errors`，继续处理其余 | 是 |

## 14. 安全相关

| 场景 | 行为 | 暴露 |
