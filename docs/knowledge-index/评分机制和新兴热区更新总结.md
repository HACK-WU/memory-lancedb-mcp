# 评分机制和新兴热区更新总结

## 核心改进

基于用户反馈，对评分机制进行了全面重新设计，并引入了新兴热区概念，解决了以下问题：

1. **历史霸占热区**：历史高分内容长期占据热区，新内容难以进入
2. **新兴热门响应慢**：新内容无法快速进入热区
3. **衰减计算开销**：避免对所有内容实时计算衰减
4. **评分算法过于简单**：之前简单的+1/-1机制无法准确反映使用频率

---

## 关键设计决策

### 1. 评分算法：基于使用密度

**旧方案**：简单的+1/-1计数
**新方案**：基于使用密度的复合评分

```javascript
最终评分 = 基础评分 × 活跃度加成
```

**基础评分公式**：
```javascript
densityScore = 100 / (1 + avgIntervalMinutes / halfLifeMinutes)
```
- `avgIntervalMinutes`：最近N次使用的平均间隔（分钟）
- `halfLifeMinutes`：半衰期，默认30分钟
- 使用间隔越短，评分越高

**活跃度加成**：
- 1小时内：评分×2.0
- 6小时内：评分×1.8
- 24小时内：评分×1.5
- 48小时内：评分×1.2
- 超过48小时：无加成（×1.0）

### 2. 边界衰减机制

**设计目标**：避免对所有内容实时计算衰减，只在内容进入热区时触发边界衰减。

**衰减流程**：
1. 保存常温区最高分 `origin_max`
2. 常温区最高分 -10
3. 热区最低分 = `origin_max`
4. 热区最高分 -10
5. 新内容进入热区

**优势**：
- 无需对所有内容计算衰减，只计算边界值
- 计算量 O(1)，性能最优
- 自动流动：热区最低分流入常温区
- 分数平衡：热区和常温区最高分都会下降

### 3. 新兴热区机制

**设计目标**：让最近频繁使用的内容能快速进入热区。

**新兴热区定义**：
- 最近48小时内使用过的内容
- 有保留席位（默认10个）
- 不受边界衰减限制

**分区结构**：
```
热区 = 新兴热区 + 历史热区
常温区 = 中等评分内容
冷区 = 低评分内容
```

**分区逻辑**：
1. 过滤达到最低分数阈值的内容
2. 识别新兴热门内容（最近48小时内使用过）
3. 先分配新兴热区席位（保留席位）
4. 再分配历史热区席位（按评分排序）
5. 分配常温区和冷区

## 参数配置

### 核心配置

```json
{
  "CONFIG": {
    "minRecordInterval": 5,      // 最小记录间隔（分钟）
    "maxRecordCount": 10,        // 最多保留的使用记录数
    "halfLifeMinutes": 30        // 半衰期（分钟）
  },
  "ACTIVITY_CONFIG": {
    "recentHours": 48            // 活跃时间窗口（小时）
  },
  "PARTITION_CONFIG": {
    "minScoreThreshold": 10,     // 最低进入热区分数
    "hotPercent": 0.3,           // 热区占比（前30%）
    "warmPercent": 0.5,          // 常温区占比（中间50%）
    "reservedEmerging": 10,      // 为新兴热门保留的席位（默认10）
    "recentHours": 48,           // "新兴热门"的时间窗口（小时）
    "minHotCount": 1,            // 最少热区数量
    "maxCounts": {
      "hot": 20,                 // 热区最大数量
      "warm": 100,               // 常温区最大数量
      "cold": 200                // 冷区最大数量
    }
  }
}
```

### 动态控制参数

所有关键参数都支持动态配置：
- `reservedEmerging`：保留席位数量（默认10）
- `recentHours`：新兴热门时间窗口（默认48小时）
- `minScoreThreshold`：最低分数阈值（默认10分）
- `hotPercent`：热区占比（默认30%）

## 性能优化

### 边界衰减 vs 实时衰减

| 方案 | 计算量 | 实时性 | 实现复杂度 |
|------|--------|--------|-----------|
| 实时衰减 | O(n) 每次查询 | 最好 | 高 |
| 边界衰减 | O(1) 每次触发 | 好 | 低 |

**边界衰减优势**：
- 只在内容进入热区时触发衰减
- 计算量 O(1)，性能最优
- 自动流动：热区最低分流入常温区
- 分数平衡：热区和常温区最高分都会下降

## 总结

新的评分机制和新兴热区设计通过以下方式解决了原有问题：

1. **基于使用密度的复合评分**：准确反映使用频率
2. **边界衰减机制**：性能最优，计算量 O(1)
3. **新兴热区保留席位**：让最近频繁使用的内容能快速进入热区
4. **自动流动**：热区最低分流入常温区，常温区最高分下降

这个设计既保持了评分机制的准确性，又解决了历史霸占热区和新兴热门响应慢的问题，特别适合代码开发场景（48小时时间窗口）。
