# 评分机制更新总结

## 核心改进

基于用户反馈，对评分机制进行了全面重新设计，解决了以下问题：

1. **历史霸占热区**：历史高分内容长期占据热区，新内容难以进入
2. **新兴热门响应慢**：新内容无法快速进入热区
3. **冷启动问题**：新内容需要很长时间才能累积到高分
4. **评分算法过于简单**：之前简单的+1/-1机制无法准确反映使用频率

---

## 关键设计决策

### 1. 评分算法：基于使用密度

**旧方案**：简单的+1/-1计数
**新方案**：基于使用密度的复合评分

```javascript
最终评分 = 基础评分 × 时间衰减因子 × 活跃度加成
```

**基础评分公式**：
```javascript
densityScore = 100 / (1 + avgIntervalMinutes / halfLifeMinutes)
```
- `avgIntervalMinutes`：最近N次使用的平均间隔（分钟）
- `halfLifeMinutes`：半衰期，默认30分钟
- 使用间隔越短，评分越高

### 2. 衰减机制

**时间衰减因子**：
```javascript
decayFactor = Math.exp(-0.693 * excessHours / decayHalfLifeHours)
```
- 衰减阈值：2小时（超过后开始衰减）
- 衰减半衰期：3天（每3天衰减一半）

### 3. 活跃度加成

**48小时时间窗口**（符合代码开发场景）：
- 1小时内：评分×2.0
- 6小时内：评分×1.8
- 24小时内：评分×1.5
- 48小时内：评分×1.2
- 超过48小时：无加成（×1.0）

### 4. 混合分区策略

**解决历史霸占热区问题**：
- **最低分数阈值**：10分（防止低分内容进入热区）
- **相对排名分区**：前30%为热区，中间50%为常温区，后20%为冷区
- **保留席位**：为新兴热门保留10个席位（可配置）

**分区逻辑**：
1. 过滤：只考虑达到最低分数阈值的内容
2. 识别新兴热门：最近48小时内使用过的内容
3. 分配热区席位：
   - 历史高分席位 = 总热区席位 - 保留席位
   - 新兴热门席位 = 保留席位
4. 填充常温区和冷区

## 参数配置

### 核心配置

```json
{
  "CONFIG": {
    "minRecordInterval": 5,      // 最小记录间隔（分钟）
    "maxRecordCount": 10,        // 最多保留的使用记录数
    "halfLifeMinutes": 30        // 半衰期（分钟）
  },
  "DECAY_CONFIG": {
    "decayThresholdHours": 2,    // 衰减阈值（小时）
    "decayHalfLifeDays": 3       // 衰减半衰期（天）
  },
  "ACTIVITY_CONFIG": {
    "recentHours": 48            // 活跃时间窗口（小时）
  },
  "PARTITION_CONFIG": {
    "minScoreThreshold": 10,     // 最低进入热区分数
    "hotPercent": 0.3,           // 热区占比（前30%）
    "warmPercent": 0.5,          // 常温区占比（中间50%）
    "reservedEmerging": 10,      // 为新兴热门保留的席位（默认10）
    "recentHours": 48,           // "新兴热门"的时间窗口（小时）
    "minHotCount": 1,            // 最少热区数量
    "maxCounts": {
      "hot": 20,                 // 热区最大数量
      "warm": 100,               // 常温区最大数量
      "cold": 200                // 冷区最大数量
    }
  }
}
```

### 动态控制参数

所有关键参数都支持动态配置：
- `reservedEmerging`：保留席位数量（默认10）
- `recentHours`：新兴热门时间窗口（默认48小时）
- `minScoreThreshold`：最低分数阈值（默认10分）
- `hotPercent`：热区占比（默认30%）

## 解决的问题

### 1. 历史霸占热区

**旧方案**：固定阈值（60分进热区），历史高分内容长期占据热区
**新方案**：相对排名 + 保留席位，新兴热门有专门席位

### 2. 新兴热门响应慢

**旧方案**：需要累积足够分数才能进热区
**新方案**：活跃度加成 + 保留席位，最近频繁使用的内容能快速进热区

### 3. 冷启动问题

**旧方案**：新内容评分从0开始，需要很长时间累积
**新方案**：新内容首次使用就有2分（1分基础×2.0活跃度加成），频繁使用可快速上升

### 4. 评分算法过于简单

**旧方案**：简单的+1/-1计数，无法区分使用频率
**新方案**：基于使用密度的复合算法，准确反映使用频率

## 总结

新的评分机制通过以下方式解决了原有问题：

1. **复合评分算法**：基于使用密度 + 时间衰减 + 活跃度加成，准确反映使用频率
2. **混合分区策略**：相对排名 + 保留席位，平衡历史积累和新兴热门
3. **灵活参数配置**：所有关键参数支持动态调整，适应不同场景
4. **完善的异常处理**：防刷分、数据损坏恢复、并发控制等

这个设计既保持了评分机制的准确性，又解决了历史霸占热区和新兴热门响应慢的问题，特别适合代码开发场景（48小时时间窗口）。
