## 简介

memory-lancedb-mcp 提供了强大的环境变量配置系统，支持在 YAML 配置文件中使用 `${ENV_VAR}` 语法进行变量引用，并提供了多种环境变量覆盖机制。本文档将详细介绍环境变量的使用方法、工作机制和最佳实践。

## 环境变量语法

### 基本语法

项目支持在 YAML 配置文件中使用 `${VARIABLE_NAME}` 语法来引用环境变量。这种语法在配置解析时会被自动替换为对应的环境变量值。

### 语法特点

- **变量格式**：`${VARIABLE_NAME}`
- **大小写敏感**：变量名区分大小写
- **空白字符处理**：变量名会自动去除首尾空白字符
- **嵌套支持**：可以在任何字符串位置使用

### 在配置文件中的应用

在默认配置模板中，多个关键配置项使用了环境变量语法：

```yaml
embedding:
  apiKey: "${OPENAI_API_KEY}"
  model: "text-embedding-3-small"
  baseURL: "https://api.openai.com/v1"

# 可选: 智能提取配置
smartExtraction:
  enabled: true
  apiKey: "${OPENAI_API_KEY}"
```

## 配置文件路径覆盖

### MEM_CONFIG_PATH 环境变量

`MEM_CONFIG_PATH` 是最重要的环境变量，用于完全覆盖默认的配置文件查找逻辑。

### 查找优先级

配置文件的查找遵循严格的优先级顺序：

1. **MEM_CONFIG_PATH 环境变量**（最高优先级）
2. **用户主目录配置**：`~/.config/memory-mcp/config.yaml`
3. **当前目录配置**：`./config.yaml`
4. **默认路径**（可能不存在）

### 路径解析逻辑

```mermaid
flowchart TD
Start([开始]) --> CheckEnv["检查 MEM_CONFIG_PATH 环境变量"]
CheckEnv --> HasEnv{"MEM_CONFIG_PATH 是否设置？"}
HasEnv --> |是| UseEnv["使用环境变量指定的路径"]
HasEnv --> |否| CheckHome["检查 ~/.config/memory-mcp/config.yaml"]
CheckHome --> HomeExists{"文件是否存在？"}
HomeExists --> |是| UseHome["使用用户主目录配置"]
HomeExists --> |否| CheckCWD["检查 ./config.yaml"]
CheckCWD --> CWDExists{"文件是否存在？"}
CWDExists --> |是| UseCWD["使用当前目录配置"]
CWDExists --> |否| UseDefault["使用默认路径可能不存在"]
UseEnv --> End([结束])
UseHome --> End
UseCWD --> End
UseDefault --> End
```

## 数据库路径动态覆盖

### 覆盖机制

数据库路径的覆盖发生在配置加载过程的最后阶段：


## 环境变量扩展机制

### 扩展函数实现

环境变量扩展是一个递归处理过程，支持复杂的嵌套结构：

```mermaid
flowchart TD
Start([expandEnvVars]) --> CheckType{"参数类型检查"}
CheckType --> |字符串| ReplaceVars["替换 ${VAR_NAME} 引用"]
CheckType --> |数组| MapArray["递归处理数组元素"]
CheckType --> |对象| IterateObj["遍历对象属性"]
CheckType --> |其他| ReturnVal["直接返回原值"]
ReplaceVars --> CheckEnvVar{"检查环境变量是否存在"}
CheckEnvVar --> |存在| ReturnEnv["返回环境变量值"]
CheckEnvVar --> |不存在| WarnUser["发出警告并返回空字符串"]
MapArray --> Recurse["递归调用 expandEnvVars"]
IterateObj --> Recurse
Recurse --> ReturnExpanded["返回扩展后的值"]
ReturnEnv --> ReturnExpanded
ReturnVal --> ReturnExpanded
```

**图表来源**
- [config.ts:135-157](file://src/config.ts#L135-L157)

### 处理流程详解

1. **字符串处理**：使用正则表达式 `/\$\{([^}]+)\}/g` 查找所有变量引用
2. **变量验证**：检查环境变量是否存在于 `process.env`
3. **值替换**：将变量引用替换为对应的环境变量值
4. **警告机制**：未设置的变量会发出警告但不会导致配置失败

### 支持的数据类型

- **字符串**：主要处理目标，支持变量替换
- **数组**：递归处理每个元素
- **对象**：递归处理每个属性值
- **原始类型**：直接返回（数字、布尔值等）

### 性能考虑

- **时间复杂度**：O(n)，其中 n 是配置树中字符串节点的数量

## 警告机制与回退行为

### 警告机制

当引用的环境变量未设置时，系统会发出详细的警告信息：

```javascript
console.warn(`[mem:config] Warning: env var ${varName} is not set`);
```

警告信息包含：
- **组件标识**：`[mem:config]`
- **变量名称**：具体的环境变量名
- **上下文信息**：指示变量未设置

### 回退行为

对于未设置的环境变量，系统采用安全的回退策略：

1. **字符串替换为空字符串**：`${UNSET_VAR}` → `""`
2. **不影响配置加载**：配置文件仍可正常加载
3. **运行时验证**：在关键功能处进行验证

### 错误处理策略

```mermaid
flowchart TD
Start([配置加载]) --> LoadFile["读取配置文件"]
LoadFile --> ParseYAML["解析 YAML"]
ParseYAML --> ExpandEnv["扩展环境变量"]
ExpandEnv --> CheckRequired["检查必需字段"]
CheckRequired --> HasAPIKey{"embedding.apiKey 存在？"}
HasAPIKey --> |是| Success["配置有效"]
HasAPIKey --> |否| CheckTemplate["检查是否为模板变量"]
CheckTemplate --> |是| WarnMissing["发出警告但继续"]
CheckTemplate --> |否| Error["抛出配置错误"]
WarnMissing --> Success
```
